> Difference b/w Encapsulation and Abstraction:

Encapsulation:
1. Encapsulation is the process of combining data and methods in class
2. Encapsulation is the process of hiding the sensitive data from the user,
3. And only providing the access to the data through public methods.
4. It is achieved by using private and protected access modifiers.

Example (Employee class):
=========================
class Employee:
    def __init__(self, name, salary):
        self.__name = name # private variable
        self.__salary = salary # private variable

    def get_name(self):
        return self.__name

    def get_salary(self):
        return self.__salary

emp_obj = Employee("mike", 50000)
print(emp_obj.get_name()) # mike
print(emp_obj.get_salary()) # 50000
print(emp_obj.__name) # AttributeError: 'Employee' object has no attribute '__name
print(emp_obj.__salary) # AttributeError: 'Employee' object has no attribute '__salary


Abstraction:
1. Abstraction is the process of hiding the implementation details from the user,
2. And only providing the functionality to the user.
3. It is achieved by using abstract classes and interfaces.
4. Hide implementation details and only show essential functionality

Example (Employee class):
=========================
abc- module
ABC- class
@abstractmethod- decorator

# Scenario:
 - we define a general Employee class
 - But we don't define how salary is calculated
 - Eac employee type will implement its own logic

from abc import ABC, abstractmethod

class Employee(ABC):

    def __init__(self, name):
        self.name = name

    # it defines but it does not implement it.
    @abstractmethod
    def calculate_salary(self):
        pass

# emp = Employee('Mike')
# you will get TypeError: can't intantiate abstract class
# here use konws, emp.calculate_salary()
# but user does not know is it monthly, is it hourly, what formula used
# implementation is hiden


class FullTimeEmployee(Employee):

    def __init__(self, name, monthly_salary):
        super().__init__(name)
        self.monthly_salary = monthly_salary

    def calculate_salary(self):
        return self.monthly_salary

class PartTimeEmployee(Employee):
    def __init__(self, monthly_salary, hourl_rate, hours):
        super().__init__(name)
        self.hourl_rate = hourl_rate
        self.hours = hours

    def calculate_salary(self):
        return self.hourl_rate * self.hours


emp1 = FullTimeEmployee('Mike', 5000)
emp2 = PartTimeEmployee('Sachin', 20, 100)

print(emp1.name, emp1.calculate_salary())
print(emp1.name, emp1.calculate_salary())



Encapsulation: Hiding data, uses private variables, protects state
Abstraction: Hiding logic, uses abstract methods, define structure without implementation
