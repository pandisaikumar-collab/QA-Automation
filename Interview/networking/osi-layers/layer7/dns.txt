> DNS: Domain Name System:
 1. DNS converts domains names (like www.example.com) into IP addresses
 2. DNS operates at Layer 7 (Application Layer) of the OSI model
 3. DNS uses UDP port 53 for queries and responses

 > DNS resolution process:
  1. A user types a domain name into their web browser
  2. The browser checks its cache for the corresponding IP address
  3. If not found, the browser sends a DNS query to the configured DNS resolver (usually provided by the ISP)
  4. The DNS resolver checks its cache; if not found, it queries root DNS servers
  5. Root servers respond with the address of TLD (Top-Level Domain) servers (e.g., .com, .org)
  6. The resolver queries the TLD server, which responds with the authoritative DNS server for the domain
  7. The resolver queries the authoritative server, which responds with the IP address of the domain
  8. The resolver returns the IP address to the user's browser
  9. The browser uses the IP address to establish a connection to the web server and load the website

What happens if DNS resolution fails?
 Website won't open using the domain name, but IP may still work.
    Common reasons for DNS failure:
    - Incorrect DNS server settings
    - DNS server is down or unreachable
    - Domain name does not exist (NXDOMAIN)
    - Network connectivity issues

Example:
 User tries to access www.redbus.com
 1. Browser checks cache - not found
 2. Sends query to DNS resolver
 3. Resolver queries root server - gets TLD server for .com
 4. Resolver queries TLD server - gets authoritative server for example.com
 5. Resolver queries authoritative server - gets IP address
