OSI Model-7 Layers Overview:
============================
> The OSI (Open Systems Interconnection)
> Think of OSI like sending a message on Whatsapp or Accessing a website.
> It breaks down the communication process into 7 distinct layers, each with its own role.

7. Application Layer:
---------------------
 > This is where communication starts and ends.
 > It provides services directly to user applications, like web browsers and email clients.
 > This is what you directly use, it lets the user talk to the network.

  Example:
   - Opening a website in Chrome
   - Sending an email
   - Using a Whatsapp or Jira

  Protocols:
   - protocols: HTTP/HTTPS, FTP, SMTP, SNMP, DNS, REST APIs
   - services: web browsing, email, file transfer, network management

6. Presentation Layer:
----------------------
> This layer translates data between the application layer and the network format.
> It handles data encryption, compression, and translation.

  Example:
   - HTTPS encryption for secure web browsing (lock icon in browser)
   - File compression (like ZIP files)
   - Data format translation (like converting text to binary/ like JSON, XML)

  Protocols:
   - protocols: SSL/TLS, JPEG, GIF, MPEG, ASCII, EBCDIC
   - services: data encryption, data compression, data translation
   - Data encoding: (JSON/XML/ASCII)

5. Session Layer:
-------------------
> This layer manages sessions or connections between applications.
> Means layer creates, manages, and ends a session between two systems.

  Example:
   - Logging into a website (establishing a session)
   - Video conferencing (maintaining a session)
   - Like Google Meet or Zoom calls
   - Session Establishment, Session Maintenance, Session Termination
   - Authentication and Authorization

  Protocols:
    - protocols: NetBIOS, PPTP, SAP, RPC
    - services: session establishment, session maintenance, session termination

4. Transport Layer:
-------------------
> This layer ensures reliable data transfer between devices.
  (like decides how data is delivered - fast or reliable)
> It manages error detection, flow control, and data segmentation.
> It ensure complete data transfer.

  Example:
   - Downloading a file (ensuring all parts arrive correctly)
   - Streaming a video (managing data flow)
   - Like Netflix or YouTube buffering (speed is important - some data loss is acceptable)
   - File downloading (accuracy is important - no data loss)

  Protocols:
    - protocols: TCP (reliable, ordered, error-checked), UDP (faster, less reliable)
    - ports: TCP (port 80 for HTTP, port 443 for HTTPS), UDP (port 53 for DNS)
    - services: flow control, error detection, data segmentation

3. Network Layer:
-----------------
> This layer is responsible for data routing, forwarding, and addressing.
> It finds the best path to send data from source to destination across multiple networks.

  Example:
   - Accessing a website from your home network (routing data through the internet)
   - Sending data from your phone to a server in another country
   - Like Google Maps finding the best route for your trip

   Protocols:
    - protocols: IP addressing (IPv4, IPv6)
    - routing protocols: OSPF, BGP, EIGRP
    - services: logical addressing, routing, packet forwarding

2. Data Link Layer:
-------------------
> This layer handles data transfer between devices on the same network.
> Means (delivers data inside the same network)
> It manages physical addressing, error detection, and access control.

  Example:
   - Connecting to a Wi-Fi network (data link layer manages the connection)
   - Transferring files between computers on the same LAN
   - Laptop communicating with a printer over Ethernet
   - Switch forwarding data traffic within a local network

  Protocols:
   - protocols: Ethernet, MAC addressing, Switching, VLAN, ARP
   - services: physical addressing, error detection, access control

1. Physical Layer:
-----------------
> This is the lowest layer, responsible for the physical connection between devices.
> It deals with the transmission of raw bits over a physical medium (like cables, radio waves)
> It defines electrical and physical specifications for devices.

  Example:
   - Ethernet cables connecting to switches and routers
   - Wi-Fi signals transmitting data wirelessly
   - Fiber optic cables for high-speed internet
   - Bluetooth connections between devices

  Protocols:
  - protocols: Ethernet (cabling standards), Wi-Fi (IEEE 802.11), Bluetooth
   - services: a bit of transmission, physical connection, signal encoding
