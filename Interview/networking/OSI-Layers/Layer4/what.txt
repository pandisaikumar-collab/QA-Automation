> OSI layer 4 is the Transport Layer. It is responsible for
  sending data one device to another device correctly and completely.

> Key functions of the Transport Layer include:
    1. Segmentation: Divides large data into smaller segments for easier transmission.
    2. Error Detection and Correction: Ensures data integrity by checking for errors and requesting retransmission if needed.
    3. Flow Control: Manages the rate of data transmission to prevent overwhelming the receiving device.
    4. Connection Establishment and Termination: Sets up and tears down connections between devices
    5. Multiplexing: Allows multiple applications to use the same network connection simultaneously.

> Common protocols used in the Transport Layer are:

 1. Transmission control protocol (TCP): Provides reliable,
    connection-oriented communication with error checking and flow control.
 2. User datagram protocol (UDP): Offers connectionless communication with minimal overhead,
    suitable for applications that require speed over reliability, such as video streaming or online gaming.

> Examples of Transport Layer services include:
    - File Transfer Protocol (FTP)
    - Simple Mail Transfer Protocol (SMTP)
    - Hypertext Transfer Protocol (HTTP)
    - Domain Name System (DNS)
    - Voice over IP (VoIP)
    - Streaming services
