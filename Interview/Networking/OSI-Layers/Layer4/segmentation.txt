> Breaking big data into small pieces before sending.
> This is done to ensure that data can be sent efficiently and reliably over the network.
> Each segment is given a sequence number to help reassemble the data at the destination.
> Segmentation helps in error detection and correction, as smaller pieces of data are easier to manage
  and retransmit if necessary.

Example:
> Suppose you want to send a large file over the internet.
> The file is broken down into smaller segments, each with its own header containing information
  such as sequence number and checksum.
> These segments are then sent over the network to the destination.
> At the destination, the segments are reassembled in the correct order using the sequence numbers,
  and any errors are detected and corrected using the checksum information.
