> HTTPS is a secure version of HTTP (Hypertext Transfer Protocol)
> It encryption to protect data transmitted between a web server and a client (usually a web browser).
> HTTPS uses SSL (Secure Sockets Layer) or TLS (Transport Layer Security) protocols to encrypt the data.
> The main purpose of HTTPS is to ensure data integrity, confidentiality, and authentication.
> HTTPS operates on Layer 7 (Application Layer) of the OSI model.
> When a user accesses a website using HTTPS, the following process occurs:
    1. The client (web browser) sends a request to the server for a secure connection.
    2. The server responds with its SSL/TLS certificate, which contains the
       server's public key and other identifying information.
    3. The client verifies the server's certificate against a list of trusted Certificate Authorities (CAs).
    4. If the certificate is valid, the client and server establish a secure session by
       agreeing on encryption algorithms and generating session keys.
    5. All data transmitted between the client and server is encrypted using the session keys.
    > HTTPS is commonly used for online banking, e-commerce, and any website that requires secure data

Keyword: HTTPS, SSL, TLS, Encryption
