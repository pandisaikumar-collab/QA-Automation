q) what is the difference between using multiple physical
   interfaces and the router-on-a-stick method for inter-VLAN routing:

 - router and stick method nothing bug inter VLAN routing:
    1) uses a single physical interface on the router
    2) sub-interfaces are created for each VLAN (logical separation)
    3) encapsulates VLAN tags on the single interface
    4) simpler setup for small networks
    5) can become a bottleneck with high traffic


q) in the router-on-a-stick method, when using the encapsulation dot1q command,
   do we specify the VLAN ID on the interface ID:

   - yes, for VLAN traffic we specify the VLAN ID on the sub-interface:
     1) interface GigabitEthernet0/0.10
     2) encapsulation dot1q 10
     3) ip address
