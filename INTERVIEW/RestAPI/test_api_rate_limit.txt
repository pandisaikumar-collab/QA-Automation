q) how to test API rate limiting:

 - send rapid requests to the API endpoint
 - expect 429 Too Many Requests response after exceeding the limit
 - validate retry-after header for wait time

Example using Python and requests library:

import requests
import time

url = "https://api.example.com/endpoint"
headers = {
    "Authorization": "Bearer YOUR_API_KEY"
}
max_requests = 10
time_window = 60  # seconds
requests_made = 0
start_time = time.time()
for i in range(20):  # Attempt to make 20 requests
    if requests_made < max_requests:
        response = requests.get(url, headers=headers)
        requests_made += 1
        print(f"Request {i+1}: Status Code: {response.status_code}")
    else:
        elapsed_time = time.time() - start_time
        if elapsed_time < time_window:
            wait_time = time_window - elapsed_time
            print(f"Rate limit exceeded. Waiting for {wait_time:.2f} seconds.")
            time.sleep(wait_time)
        # Reset counters after waiting
        requests_made = 0
        start_time = time.time()
        response = requests.get(url, headers=headers)
        requests_made += 1
        print(f"Request {i+1}: Status Code: {response.status_code}")
